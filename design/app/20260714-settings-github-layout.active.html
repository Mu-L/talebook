配置页面 GitHub 风格双栏布局

ACTIVE 创建日期：2026-07-14 所属模块：app 需求来源：用户口头需求（附 GitHub 设置页截图）

1. 原始诉求

优化配置页面的体验，参考 GitHub 的这种功能风格来做，并且右侧的配置是完整可滚动的，滚到某个类别后 menu 就自动选中对应的栏目。

用户随需求提供了 GitHub「Settings」页面截图：左侧带分组小标题的导航菜单，右侧为完整可滚动的配置内容。

澄清确认（grill 结果）

决策点选择
右侧内容形态全部展开、连续滚动（去掉卡片折叠交互）
左侧菜单分组分组，带小标题
保存按钮底部固定悬浮条
窄屏/移动端隐藏侧栏，改为顶部下拉跳转，内容占满宽度

2. 目标

  • 将 app/pages/admin/settings.vue 从「纵向堆叠的可折叠卡片」改为「左导航 + 右滚动内容」双栏布局。
  • 所有配置分类默认展开，右侧为一条连续可滚动的长内容。
  • 左侧菜单按语义分组并带小标题；点击菜单项平滑滚动到对应分类。
  • 滚动到某分类时，左侧对应菜单项自动高亮（scroll-spy）。
  • 顶部悬浮固定标题栏（左侧页面标题 + 右侧保存按钮），滚动时始终贴在应用顶栏下方可见。
  • 窄屏折叠侧栏为顶部下拉选择，内容占满宽度。
  • 不改变任何配置项的读写逻辑与后端接口，仅重构展示层。

3. 方案

3.1 布局结构

┌───────────────────────────────────────────────┐
│ AppHeader (v-app-bar, 固定, 48px)               │
├───────────────────────────────────────────────┤
│ 悬浮标题栏(sticky top:48)  系统设置 … [保存配置] │
├────────────────────────────────┬──────────────┤
│  content (window 滚动)          │ nav (sticky) │
│   ▸ section#basicInfo           │  站点         │
│   ▸ section#bookCategories      │   基础信息 ●  │
│   ▸ section#friendshipLinks     │   书籍分类    │
│   ▸ section#userSettings        │  访问与用户   │
│   ...                            │  服务与集成   │
│                                  │  系统         │
└────────────────────────────────┴──────────────┘
  • 顶部悬浮标题栏 position: sticky; top: 48px（贴 app-bar 下方）；外层 flex 用 row-reverse：内容在左（flex: 1），导航在右（sticky; top: 108px，位于标题栏下方）。
  • 沿用页面本身随 window 滚动（layout 为 v-main > v-container），不引入独立滚动容器。
  • 每个分类渲染为 <section :id>，scroll-margin-top: 116px 抵消 app-bar + 标题栏高度；scroll-spy 判定线同步取 104 + 12。

3.2 分组

分组包含分类
站点 (site)基础信息、书籍标签分类、友情链接
访问与用户 (access)用户设置、社交网络登录、人机验证设置
服务与集成 (services)邮件服务、设备管理、书籍信息源、OPDS、WebDAV
系统 (system)高级配置项、数据库管理、SSL 证书管理、回收站管理、检查更新

分组小标题新增 i18n key：admin.settings.group.{site,access,services,system}；导航选择器标签 admin.settings.label.navSection。

3.3 Scroll-spy

  • 基于滚动位置计算：取「上缘已越过判定线（头部下方 ACTIVE_LINE）」的最后一个分类作为 activeKey。
  • 滚动监听用 requestAnimationFrame 节流；仅客户端启用，SSR 不注册监听。
  • 末尾分类无法上移到判定线：当页面确有滚动空间且已滚到底部时，直接激活最后一个分类。
  • 页面加载停在顶部，默认激活第一个分类，加载期不做计算（避免数据未就绪、页面偏短时的误判）。
  • 点击菜单项用 scrollIntoView({ behavior: 'smooth' }) 跳转，跳转期间短暂锁定 spy 防抖动。
  • 导航项过多导致侧栏内部滚动时，仅滚动侧栏本身使当前项可见，不影响页面滚动。

说明：初版曾用 IntersectionObserver，因末尾分类与加载期页面高度变化导致高亮误判，改为更直接可控的滚动位置计算。

3.5 视觉风格（GitHub 对齐）

  • 导航菜单置于右侧，配置内容在左侧；两栏之间以竖线分隔（内容区 border-right，全高），导航左对齐贴线、内容右侧留 24px 间距。
  • 导航宽 172px；分组小标题加粗（font-weight: 800），标题下方加横线（border-bottom），块与块之间留更多空白。
  • 取消分类之间的横向分隔线，仅保留每个分类标题下的下划线（GitHub 小节样式），减少视觉冗余。
  • 整体紧凑：分类标题降为 16px，通用输入控件统一 density="compact"，缩小间距，正文常规字号（14px）。

3.4 数据与交互不变

  • 保留 cards computed 作为内容与菜单的唯一数据源，新增 group 字段用于分组。
  • 删除卡片折叠状态 cardShows 与 v-expand-transition/v-show 逻辑。
  • 字段渲染、按钮、friends/devices/socials/db/ssl/update/trash 等特殊块逻辑原样保留。
  • 保存、测试邮件、数据库迁移、回收站、检查更新等方法完全不变。

4. 影响范围

  • app/pages/admin/settings.vue：模板与样式重构，脚本仅增删导航/scroll-spy 相关逻辑。
  • app/i18n/locales/zh-CN.json 、app/i18n/locales/en-US.json ：新增 group / navSection 文案。
  • app/test/e2e/settings.spec.ts ：新增导航与 scroll-spy 相关 E2E 用例（或补充现有）。
  • 后端、API、数据结构：无变化。
