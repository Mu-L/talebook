Docker 生产阶段依赖瘦身

创建日期：2026-07-16 所属模块：docker 状态：ACTIVE 需求来源：用户直接提出
生产构建改用本地预发布的 talebook/talebook-base:slim-v8.5.0；删除共享阶段重复安装的运行工具与 vim；把测试、覆盖率和 lint 工具从生产依赖中拆出，只在 Docker test 阶段安装。

原始诉求与已确认决策

原始诉求包含三项：生产环境阶段移除 vim 等无关组件；确保 pytest 等组件只放在测试阶段；基础镜像切换为 slim-v8.5.0。

决策结论依据
完整基础镜像引用talebook/talebook-base:slim-v8.5.0方案制定时用户确认该镜像仅存在于本地；2026-07-16 已发布到 Docker Hub。
共享 apt 安装层整体删除 gettext gosu procps vim 安装层已从本地 slim 镜像核实：envsubst、gosu、ps 已存在，vim 不存在。
测试工具范围ruff、pytest、pytest-cov、flake8这些工具不参与 Talebook 生产运行。
Python 元数据同步迁入 pyproject.toml 的 test optional dependencies避免项目元数据继续把测试工具声明为生产依赖。

目标与非目标

目标

  • 生产镜像不包含 vim、pytest、pytest-cov、ruff 和 flake8。
  • 测试镜像仍可运行完整 pytest、覆盖率与 lint 工具链。
  • 本地初始化命令仍一次性安装开发所需的生产和测试依赖。
  • 通过自动化测试固定 Docker 阶段边界，防止后续把测试工具重新带入生产。

非目标

  • 本次不修改 Dockerfile.base ，不负责发布 slim-v8.5.0。
  • 不改变 SPA、SSR、dev 镜像的业务行为或 Talebook 功能。
  • 不继续裁剪 slim 基础镜像内仍存在的运行时组件。

方案

文件计划改动
Dockerfile切换基础镜像；删除重复 apt 安装层；测试阶段复制并安装 requirements-test.txt 。
requirements.txt移除四项测试与 lint 工具，仅保留生产依赖。
requirements-test.txt新增测试依赖清单并保留现有版本约束。
pyproject.toml / uv.lock把相同工具迁入 test optional dependency group，并同步锁文件。
Makefilemake init 同时安装两份 requirements，保持开发体验。
tests/test_docker_stages.py静态校验镜像引用、生产/测试依赖归属和 Docker 阶段安装关系。

兼容性、发布与回滚

  • 兼容性：slim 基础镜像必须继续提供 envsubst、gosu 和 ps；本地镜像已验证，发布到 registry 时需保持同一内容。
  • 发布状态：talebook/talebook-base:slim-v8.5.0 已于 2026-07-16 发布，Docker Hub 已提供 amd64、arm64 与 arm/v7 清单，远程 CI 的镜像拉取前置条件已满足。
  • 回滚：若 slim 镜像运行能力不完整，将 Dockerfile 的基础镜像引用恢复为原标签，并恢复共享 apt 安装层；依赖清单拆分可以独立保留。
  • 安全与性能：减少生产工具和包数量可缩小攻击面与镜像体积；不新增运行时权限、端口或网络行为。
已知风险：多架构清单已发布，但本轮只在本地 arm64 完成应用构建与运行验证；其余架构仍以远程 CI 的实际拉取和构建结果为准。
