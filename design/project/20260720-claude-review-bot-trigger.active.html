Claude 自动评审允许受信任 Codex Bot 触发

创建日期：2026-07-20 所属模块：project 状态：ACTIVE 需求来源：PR #874 CI 红灯排查

原始诉求与根因

用户发现 PR #874 再次出现 CI 红灯，要求继续处理。失败日志显示，业务测试、方案检查、CodeQL 和多架构镜像均通过，唯一失败项为 claude-review。该次 synchronize 事件由仓库受信任的 codex-talebook[bot] 提交评审修复触发，而 anthropics/claude-code-action@v1 默认拒绝所有 bot，因而在真正执行评审前退出。

目标与边界

  • 允许仓库自己的 codex-talebook[bot] 触发 Claude PR 自动评审。
  • 不使用通配符，不允许其他 GitHub App 或 bot 获得相同能力。
  • 不扩大 workflow 的 GitHub Token 权限，不改变业务代码和生产运行行为。
  • 保留现有对 claude[bot] 的 job 级排除，避免 Claude 自身提交形成评审循环。

方案

位置修改安全约束
.github/workflows/claude-code-review.yml为 Claude Action 增加 allowed_bots: 'codex-talebook[bot]'。显式单账号白名单；禁止 *。
tests/test_claude_review_workflow.py固定 action 版本、白名单值和无通配符约束。防止后续回退成默认拒绝或无边界放开。

测试结果

本地验证通过：契约测试完成红—绿循环，workflow 语义、Python lint 和完整 Calibre 容器测试均通过。真实 bot actor 只能由 GitHub 事件产生，推送后继续观察远端自动评审结果。
