01 · 已确认事实与边界

  • 两次失败的 Codex / Respond to maintainer request 都在 Resolve request context 读取 master:.github/codex/scripts/codex_progress_reporter.py 时返回 404。
  • 资源 404 发生在 🚀 reaction 与状态评论之前，失败请求没有及时向用户暴露；最终评论还包含 Validation、Published commit、Not published 等英文。
  • CodeQL 唯一行级告警位于测试用例的 https://example.com 字面量，不是产品 URL 处理逻辑。
  • GitHub 官方上下文提供 github.workflow_sha，其定义是“workflow 文件所在提交 SHA”；它可以把外部受信任资源与实际执行的 workflow 固定为同一不可变版本。
  • actions/create-github-app-token@v3 官方当前推荐 client-id，app-id 只是兼容输入；actionlint 的元数据提示属于误报，不改回旧输入。
  • 仓库当前没有 ruleset；branch protection API 返回 404（可能未启用或当前身份不可见），因此本任务不把“无强制检查”当作完成，而以 PR 上所有非预期失败检查清零为标准。
  • 用户此前明确要求更新当前 PR 分支，因此不新建 worktree；当前分支不是 master，继续在 codex/digital-employee-workflow 上修复。
不变量任何 sandbox 探测失败都必须在 Codex 启动前终止；外部 PR head 不得决定 Prompt 或进度桥接器内容；发布 Token 仍只能在 publish gate 通过后创建。
