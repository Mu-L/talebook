3. 官方行为与方案

anthropics/claude-code-action@v1 是 composite action，默认自行准备 Bun 和 Claude Code，但不会安装 Talebook 的 Calibre、Python、Node 或项目依赖。GitHub job container 会让普通 run step 与 composite action step 在指定容器中执行，因此 dev 镜像负责业务环境，Claude Action 继续负责自身运行时。

现有权限与发布门禁
  • 两个 job 都设置 defaults.run.shell: bash 和 root 用户，避免 composite/run shell 与工作区权限差异。
  • Claude 不设置 path_to_claude_code_executable 或 path_to_bun_executable；allowed tools 扩展到仓库实际使用的 make、npm、Node、Python、pytest 与 ruff。
  • Codex 容器增加 --security-opt seccomp=unconfined 以允许镜像内 bubblewrap 创建 user namespace，但不使用 privileged 或 Docker socket。
  • Codex 删除 runner 侧 setup-python、pip、npm 全局安装与 AppArmor 改写；在恢复模型凭据前验证环境、sandbox 并同步依赖。
  • 两个 job 对当前 checkout 继续执行 make init 与 npm --prefix app ci，避免镜像构建时的依赖替代目标分支 lockfile。

4. 测试缝与 TDD 顺序

公开测试缝是 workflow job contract：容器、shell、工具探测、依赖同步顺序以及 agent 启动前的失败关闭行为。测试不依赖 Action 内部实现。

  1. 先新增 Claude job contract 测试，在裸 runner 配置上确认红灯；最小迁移 Claude 后转绿。
  2. 再把 Codex 现有运行时安装契约改写为 dev container 契约，确认红灯；最小迁移 Codex 后转绿。
  3. 运行相关 pytest、ruff、actionlint、设计检查和完整 Docker 测试。
  4. 用本地已构建的 dev 镜像做等价容器命令验证；不保留临时 smoke workflow。

5. 风险与回滚

  • 浮动标签漂移：仅 master 可发布，job 启动时打印工具版本；镜像缺工具则在 agent 获取凭据前失败。
  • Claude composite action 兼容：使用 Debian/glibc、bash、curl 与 CA 基线，并保留 Action 默认 Bun/Claude 管理。
  • Codex sandbox：只放宽容器 seccomp，内层 permission profile 与凭据隔离不变；sandbox 自检失败直接退出。
  • 回滚：移除两个 job 的 container/defaults 与同步步骤，恢复 Codex runner 安装步骤；镜像本身继续可供开发使用。
