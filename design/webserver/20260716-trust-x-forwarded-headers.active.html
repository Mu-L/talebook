site_url 信任反向代理头

ACTIVE 待实现 创建日期：2026-07-16 所属模块：webserver 需求来源：Issue #498（"希望能够增加设置外网 ip+port 的功能"）

1. 原始诉求

希望能够增加设置外网 ip+port 的功能，以便在发送激活邮件等操作时，正确地发送外网地址。

当前 self.site_url = self.request.protocol + "://" + host（webserver/handlers/base.py:194）直接采用请求的 Host 头。Nginx / Caddy / Traefik 反向代理后，Tornado 看到的协议是 HTTP、host 是容器端口，发出去的激活邮件链接就会指向 http://192.168.x.x:8080/...，用户在公网无法点击。

2. 目标

  • 按业界惯例：仅当请求来自可信代理（由 TRUSTED_PROXIES 配置）时，信任 X-Forwarded-Proto 和 X-Forwarded-Host。
  • 支持显式覆盖：新增 SITE_URL 配置，优先级最高（避免误用反代头）。
  • 不改任何业务调用方——继续用 self.site_url。

3. 配置项

键类型默认说明
SITE_URLstr空显式覆盖，例如 https://books.example.com。留空则按反代头或请求头推断。
TRUSTED_PROXIESlist[str]["127.0.0.1", "::1"]可信代理 IP / CIDR。仅这些 IP 来的请求才会被允许覆盖。

两个键加入 webserver/handlers/admin.py 的 KEYS 白名单，写入 /data/books/settings/auto.py。

4. 解析逻辑

修改 webserver/handlers/base.py:192-194 的 site_url 计算：

def _compute_site_url(self):
    explicit = CONF.get("SITE_URL", "").strip()
    if explicit:
        return explicit.rstrip("/")

    remote = self.request.remote_ip or ""
    trusted = any(_cidr_match(remote, cidr) for cidr in CONF.get("TRUSTED_PROXIES", []))
    if trusted:
        proto = self.request.headers.get("X-Forwarded-Proto", self.request.protocol).split(",")[0].strip()
        host = self.request.headers.get("X-Forwarded-Host", self.request.host).split(",")[0].strip()
        return f"{proto}://{host}"

    return f"{self.request.protocol}://{self.request.host}"

_cidr_match 用 ipaddress.ip_network 实现一次，支持单 IP 和 10.0.0.0/8。

6. 边界

  • X-Forwarded-* 取逗号分隔的第一个值（最接近客户端的反代）。
  • 反代链超过一层时（如 CDN + 自建 Nginx），仅信任直接相邻的那层；CDN 头部由配置决定。
  • 不记录 / 不告警 site_url 被改写；调试时通过日志打印 remote_ip + 解析结果即可。

8. 风险

误把 TRUSTED_PROXIES 设为 0.0.0.0/0 会让任意客户端伪造 X-Forwarded-*。需要在 README / 安装引导里强调该选项的信任模型。
