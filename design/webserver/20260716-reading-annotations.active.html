阅读笔记 / 高亮后端同步设计

ACTIVE 待实现 创建日期：2026-07-16 所属模块：webserver（主） + candle-reader（外部仓库） 需求来源：Issue #181（"能否支持笔记功能"）

1. 原始诉求

希望支持笔记功能。

candle-reader 已支持高亮和本地笔记，但当前只在浏览器 localStorage 中保存。同一用户换设备或清缓存即丢失，也无法跨端同步。需要 Talebook 后端提供持久化和多端同步接口。

2. 目标

  • Talebook 后端提供"高亮 + 笔记"的 CRUD 接口，存储在服务端。
  • candle-reader 通过新接口读写，跨设备登录后能同步。
  • 支持同一本书同一位置的幂等更新（按 client_id 去重）。
  • 后端不做冲突合并：按更新时间后写覆盖（v1 简单优先）。

3. 数据模型

字段类型说明
idINTEGER PK主键
reader_idINTEGER FK→readers.id所属用户
book_idINTEGERCalibre 书籍 id
client_idVARCHAR(64)阅读器生成的 UUID，幂等键
kindVARCHAR(16)highlight / note / bookmark
cfiVARCHAR(255)EPUB CFI（PDF 用页码字符串）
textTEXT NULL高亮文本 / 笔记内容
colorVARCHAR(16) NULL前端颜色标识，如 #ffff00
chapterVARCHAR(255) NULL章节名（前端展示用）
created_atDATETIME本地首次创建
updated_atDATETIME最近修改

唯一约束：(reader_id, book_id, client_id)。便于阅读器幂等 upsert。

新表随启动自动迁移（compare_and_migrate 已支持）。

4. API

方法路径权限说明
GET/api/book/<id>/annotations登录 + 书籍可读列出当前用户在该书的所有笔记 / 高亮，按 cfi 排序
POST/api/book/<id>/annotations登录upsert：按 client_id 存在则更新，否则插入
PUT/api/book/<id>/annotations/<ann_id>owner更新（仅 text / color / chapter）
DELETE/api/book/<id>/annotations/<ann_id>owner删除

所有接口沿用 @js + @auth 装饰器；权限校验参考现有 BookReadingProgress（webserver/handlers/book.py）。

5. candle-reader 侧改动（外部仓库）

本设计只覆盖 Talebook 端。candle-reader 侧需要：

  1. 新增一个 TalebookAnnotationSync provider，把 localStorage 的笔记在打开时同步到后端、关闭前批量 upsert。
  2. 登录后从 GET /api/book/<id>/annotations 拉取本地没 client_id 的旧条目，补写 client_id 后再 upsert。
  3. 删除操作同步调用 DELETE。
candle-reader 改造不在本仓库范围，需在 candle-reader 仓库单独提 PR。本设计只锁定后端接口契约，确保双方独立推进也能对齐。

6. 边界

  • 不支持实时协作（多人同时编辑同一笔记）。v1 用 last-write-wins。
  • 不支持 PDF 笔记的位置信息精确匹配，只存页码字符串供前端跳转。
  • 不导出 / 不导入 EPUB 注释（避免与 EPUB spec 复杂对齐）。
  • 不暴露给他人；管理员也不可见其他用户笔记。
