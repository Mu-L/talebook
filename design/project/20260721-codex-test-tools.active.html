维护者指出 Codex runner 未安装 ruff 与 pytest

原始诉求与问题

Codex 在 .github/workflows/codex.yml 的 runner 中执行维护任务时，会把需要运行的 ruff 或 pytest 验证记录为“未执行”，原因是 runner 没有安装对应命令。当前 workflow 只安装 Codex CLI 和 sandbox 组件，没有准备仓库声明的 Python 测试工具。

目标与边界

  • Codex 启动前，runner 必须能执行 ruff 和 python3 -m pytest。
  • 工具版本继续由 requirements-test.txt 管理，不在 workflow 中重复维护版本。
  • 依赖清单必须来自不可变的 github.workflow_sha，不得安装 PR head 可修改的清单。
  • 安装发生在可信 workflow 步骤中，不扩大 Codex sandbox 权限，不暴露凭据。
  • 本次不改变常规 ci.yml、生产镜像或 Docker 测试阶段。

方案

上下文解析步骤从不可变的 github.workflow_sha 读取受信任 requirements-test.txt，保存到 runner 临时目录；不得直接安装 PR head 中的依赖清单。checkout 后新增 Python 环境准备与测试工具安装步骤，安装临时目录中的受信任清单，随后运行版本命令快速失败，从而把依赖准备错误暴露在 Codex 启动前。
