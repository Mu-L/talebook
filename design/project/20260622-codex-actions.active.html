Codex GitHub Actions 配置

让维护者从 issue 与 PR 触发受限的 Codex 任务，同时隔离未信任代码、登录态与写权限

状态 · ACTIVE原日期 · 2026-06-22迁移 · 2026-07-13
TL;DR
.github/workflows/codex.yml 接受维护者通过 @codex 或 /codex 发起任务。workflow 只 checkout 可信 master，将 PR diff 作为文本交给 Codex；顶层 token 已声明 contents: write，但 checkout 不持久化凭据，agent 环境排除 GitHub token，当前脚本也不执行 commit 或 push，修改仍以 patch artifact 交付。

01原始诉求

需要让仓库维护者在 issue 评论、PR 评论、PR review 和 review thread 中直接触发 Codex 协助分析或修改代码，同时避免 privileged comment workflow 执行未信任 PR 代码或泄露 ChatGPT 登录态。

02目标

  • 只允许具有 write、maintain 或 admin 权限的评论者触发。
  • PR 上下文可被分析，但 PR head 与 merge ref 不进入带 secret 的执行环境。
  • Codex 只能在 workspace-write 沙箱中修改工作区，agent 阶段关闭网络并排除仓库 token。
  • 当前不自动 push；变更作为 artifact 输出，由维护者审阅。

03方案

Secret 配置

仓库 secret CODEX_AUTH_JSON 保存本地 ~/.codex/auth.json 的完整内容。若登录态默认存于系统 keychain，先在 ~/.codex/config.toml 配置文件凭据存储，再重新登录。

cli_auth_credentials_store = "file"

运行时边界

控制点约束
认证文件写入 runner 临时目录，权限 600，不进入仓库工作区。
安装顺序先安装 Codex CLI，再恢复认证文件，避免依赖安装接触登录态。
代码来源始终 checkout 可信 master；PR files 与 diff 通过 API 读取。
Agentworkspace-write 沙箱，运行阶段无网络。
Workflow token顶层声明 contents: write、issues: write 与 pull-requests: write。
Agent Git 凭据checkout 设置 persist-credentials: false；agent 子进程环境排除 GITHUB_TOKEN 与 GH_TOKEN。
环境变量排除常见 token、secret、key，避免子进程继承。
输出脚本没有 commit 或 push；有 diff 时上传 artifact 并提示维护者。
长期预留
当前 contents: write 比 patch-only 行为所需权限更宽，应优先评估收紧为 read。若未来允许直接向 PR 分支推送，应把最小权限 token 和推送动作放入独立 job，并重新处理 fork、分支保护与未信任输入边界。

04当前实现核对

项目2026-07-13 状态
触发与授权issue 评论、review 评论和 review 均支持；先按 author association 过滤，再调用 API 确认 write、maintain 或 admin。
未信任代码隔离固定 checkout master；PR 文件和 diff 通过 GitHub API 写入临时文件，只作为提示词文本。
Codex 沙箱Codex CLI 版本固定为 0.141.0；workspace-write、网络关闭、认证文件位于 runner 临时目录。
变更交付用 git diff --binary 生成 patch，artifact 保留 7 天，并在评论中提示下载；没有自动提交或推送步骤。
权限差异原文“无写权限”不准确。workflow token 拥有 contents write，但当前没有向 agent 暴露持久化 Git 凭据。
