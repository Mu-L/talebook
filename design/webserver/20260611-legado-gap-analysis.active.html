Legado 书源引擎兼容性差距分析

以实际分页故障为入口，系统对比规则语法、URL、书源字段、工作流和 JS 扩展边界

状态 · ACTIVE原日期 · 2026-06-11对比 · Legado 2026-05迁移 · 2026-07-13
TL;DR
m.jhsssd.com 目录只得到约 25 章的根因是 nextTocUrl: "option@value" 的多 URL 语义不兼容：Legado 会逐个抓取全部分页 URL，Talebook 却把它们拼成一个无效 URL。P0 围绕列表分页、章节去重、索引语义和 URL 占位完成兼容；WebView、书源登录、网络类 JS 与非文本书源明确不移植。

01原始诉求

用户反馈某书源目录实际有 1600 余章，但 Talebook 只能读取约 25 章。需要定位直接故障，并以 Legado 2026-05 为参照，对 webserver/services/booksource/ 的兼容性做全面盘点，给出本次实施范围与后续路线。

02目标

  • 修复 nextTocUrl 与 nextContentUrl 的多 URL 列表语义。
  • 按真实使用价值划分 P0、P1、P2，而不是追求对 Android 客户端的全量复刻。
  • 明确服务端环境不能或不应承接的功能，尤其是 SSRF、登录态与浏览器内核风险。
  • 保留可扩展接口，使纯计算 JS、规则切分和书源限速可按需继续兼容。

04兼容性矩阵

已支持 · 部分支持 · 未支持。标记“本次”的项目属于 2026-06-11 的 P0 实施。

A. 规则语法

#能力状态语义与影响
A1class. / id. / tag.已支持转换为 CSS 选择器。
A2text.XXX本次支持按自有文本筛选元素；旧实现误当 tag。
A3children已支持直接子元素。
A4单索引与负索引已支持.N / .-1。
A5排除索引本次支持!N / !0:3；内置种子源在用。
A6多索引本次支持.0:3:7，存量书源常见。
A7区间、步长、反向本次支持[1,3:5,-1] 与 [-1:0]。
A8CSS / XPath / JSON / 默认模式已支持JSONPath 使用 jsonpath-ng，与 Jayway 有少量方言差异。
A9@@ 强制默认模式未支持罕用，低优先级。
A10AllInOne 正则未支持冒号开头规则与分组引用；存量占比较低。
A11&& / || / %%部分当前朴素 split，选择器或 JS 内分隔符可能误切。
A12尾部正则替换已支持##p##r。
A13replaceFirst本次支持##p##r###。
A14@put部分仅支持整条规则形式，不支持任意位置与标准 JSON 全语义。
A15@get已支持读取变量。
A16双花括号模板部分变量与子规则已支持；JS 表达式和 JSON 模板不完整。
A17规则级 JS 流水线未支持仅支持尾部单段 @js:，quickjs 受限执行。
A18HTML 实体反转义部分BeautifulSoup 解析时已处理一部分。
A19html 提取剔除 script/style未支持影响较低。

B. URL 规则

#能力状态说明
B1URL 与 options 拆分已支持url,{options}。
B2method / charset / headers / body已支持核心请求选项。
B3分页占位本次支持<a,b,c> 按 page 选段。
B4retry未支持低成本，可后补。
B5URL 后处理 JS未支持依赖 JS 运行时增强。
B6字节返回类型未支持文本书源不需要图片/字体字节流。
B7WebView / webJs / delay不移植服务端无浏览器内核。
B8POST 表单 / JSON / multipart部分当前原样发送，多数站点可用。
B9GET charset 与 escape部分依赖 requests 默认行为。
B10书源自带 proxy未支持服务端代理应由全局策略管理。
B11按源并发率未支持当前只有全局超时。
B12CookieStore 持久化部分Session 进程内保存，不落库。
B13JS 生成 URL / result 引用未支持依赖规则级 JS。
B14data URI未支持主要用于图片。

C2header JSON已支持header 为 JS 时回退默认头。
C3searchUrl / exploreUrl已支持包含分类解析。
C4书源登录不移植多用户服务端难以安全隔离书源登录态。
C5jsLib未支持依赖完整 JS 运行时。
C6concurrentRate未支持与 B11 相同。
C7coverDecodeJs未支持罕用。

C9checkKeyWord未支持书源体检可后补。
C10UI / 排序字段不需要variableComment、customOrder、respondTime 属于客户端 UI。

D. WebBook 工作流

#能力状态说明
D1nextTocUrl 多 URL本次支持本次用户问题根因；列表逐个抓取。
D2nextTocUrl 单 URL 循环本次完善逐页跟进并以 seen 防环。
D3nextContentUrl 列表语义本次支持复用 D1/D2 队列模型。
D4章节去重本次支持保留靠后重复项，剔除页首“最新章节”块。
D5目录正序 / 倒序前缀本次支持- 与 +。
D6目录页数上限调整默认 30 提升到 1000，Legado 本身不设上限。
D7卷标与 URL 回退未支持影响有卷结构的保存排版。
D8isPay未支持当前只有 isVip。
D9formatJs未支持依赖 JS。
D10更新前 JS / 动态目录 URL未支持依赖 JS 运行时决策。
D11正文页标题规则未支持可后补。
D12正文 replaceRegex部分按 pattern/replacement 列表处理，覆盖多数场景。
D13翻页遇下一章即停未支持避免把下一章并入当前章。
D14付费 / 图片处理不移植payAction、imageStyle、imageDecode 超出职责。
D15空搜索按详情页解析未支持依赖 C8。

E. JS 扩展

纯计算类 · 可安全移植

Base64 / hex、时间格式、URI 编码、HTML 格式化、章节数字、UUID、字节转换和简繁转换。

网络类 · 默认不移植

ajax、get/post/head、download、cookie 等会赋予导入书源访问服务器内网的能力，形成 SSRF。

设备环境类 · 不移植

WebView、浏览器、文件系统、字体反爬、toast、androidId 等属于 Android 客户端环境。

05方案与长期边界

P0 已实施

  1. D1/D2/D3：增加 get_string_list()，目录与正文改为队列翻页，兼容单链接与多链接，URL 去重防环。
  2. D4：章节按 name 与 URL 去重，保留靠后出现项。
  3. D5：支持 chapterList 的正序、倒序前缀。
  4. D6：BOOKSOURCE_MAX_TOC_PAGES 默认从 30 提升到 1000。
  5. A2：text.XXX 按自有文本筛选元素。
  6. A5/A6/A7：支持排除、多索引、区间、步长和反向索引。
  7. A13：支持 replaceFirst。
  8. B3：支持 URL 分页占位。

明确不移植

项目原因
WebView 渲染服务端无浏览器内核，引入 headless 浏览器的成本与攻击面不成比例。
书源登录Talebook 是多用户服务端，书源级登录态属于单用户客户端模型。
网络类 JS导入的书源 JSON 可在服务器内网发任意请求，构成 SSRF；未来若支持必须有 URL 白名单、禁内网、限流且默认关闭。
音频、图片、文件源Talebook 网络书库定位为文本小说。
字体解密与付费购买反爬军备和付费场景超出服务端书库职责。

后续路线

P1：A11 平衡切分、A16 纯计算 JS、E1 纯计算扩展、B4 retry、C8/D15 bookUrlPattern、C9 checkKeyWord、D7 卷标、D11 正文标题、D13 翻页停止条件。

P2：A10 AllInOne 正则、A14 put 增强、B8/B9 编码细化、B11/C6 并发率、B12 cookie 落库，以及依赖 JS 运行时升级决策的 D9/D10。
