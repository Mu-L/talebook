4xx / 5xx 统一错误页设计

ACTIVE 待实现 创建日期：2026-07-16 所属模块：app 需求来源：features.md（"站点美化 → 4xx 页面、5xx 页面"）

1. 原始诉求

完善 4xx 和 5xx 错误页面，避免出现 Tornado 默认的丑陋堆栈页。

当前 Nuxt SSR 模式下，访问 /book/999999（不存在的书籍）或 /api/... 直接返回 404 HTML 时，前端未做拦截，会落到 Nuxt 默认错误页或直接渲染空白。需要一个统一页面承接所有 4xx / 5xx 状态。

2. 目标

  • 新增 app/error.vue，按 HTTP 状态码显示对应文案与建议操作。
  • 支持明 / 暗主题（继承 useMainStore().theme）。
  • i18n 完整覆盖中英文。

3. 页面结构

状态标题（zh-CN）建议操作
400请求格式有误检查 URL 或返回上一页
401需要登录跳转登录页
403没有访问权限联系管理员
404找不到页面返回首页 / 搜索
500服务器内部错误稍后再试，必要时查看日志
502 / 503 / 504服务暂不可用稍后再试
其他出现错误（status = ...）返回首页

4. 实现要点

  • 使用 Nuxt 4 内置的 error.vue 约定，根级放置。
  • 通过 defineProps<{ error: { statusCode, statusMessage, message } }>() 接收错误。
  • 不展示后端堆栈（避免信息泄露）。管理员模式下可显示简短 message。
  • 提供"返回首页"和"返回上一页"按钮；401 自动跳 /login?redirect=...。

5. i18n key

新增 error.title.{400|401|403|404|500|502|503|504|other} 与 error.action.{back|home|login}。

6. 边界

  • 不影响 /api/... 的 JSON 错误响应（仍由 Tornado 装饰器返回 JSON）。
  • 404 资源请求（/get/xxx.png）保持浏览器默认行为，不渲染 SPA 错误页。
  • 不要让错误页本身抛错（避免递归）。
