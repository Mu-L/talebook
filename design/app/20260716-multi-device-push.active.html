多设备批量推送 UI 设计

ACTIVE 待实现 创建日期：2026-07-16 所属模块：app 需求来源：Issue #293（"发送到 kindle 能否多个邮箱"）

1. 原始诉求

发送到 kindle 能否多个邮箱？

后端 UserDevices（webserver/handlers/user.py:540）已支持每个用户维护多台设备（多个 Kindle 邮箱、多个 IP:PORT 自定义设备）。但书籍详情页的"推送到设备"对话框仍是单选下拉（app/pages/book/[bid]/index.vue:1305），一次只能推一本到一台设备。

2. 目标

  • 详情页推送到设备对话框改为多选 checkbox，可同时勾选多个目标。
  • 后端 BookSendToDevice 改为循环处理（已有循环雏形，需确认）。
  • 一次请求内并行推送，单个失败不影响其他设备；UI 给出每个设备的结果。

3. 后端改动（最小）

webserver/handlers/book.py 的 BookSendToDevice.post：

  • 请求体改为 {device_type: [name1, name2, ...]} 或 {devices: [...]}（多对象数组）。
  • handler 内 for device in devices 循环，沿用现有 _send_to_other_device / 邮件分支。
  • 单设备失败捕获异常，记录 device_name -> error，最终返回 {"err": "ok", "results": [{"name": ..., "ok": true}, ...]}。
  • 不引入异步队列：单本 ≤ 5 台设备的推送在几秒内完成；用户已有推送历史可查（push_history）。

4. 前端改动

位置改动

推送按钮文案"推送到 2 台设备"（动态显示数量）
请求体组装把勾选设备构造成数组提交
结果展示成功 / 部分失败分别 toast，失败项单独列在 dialog 底部

5. 边界

  • 用户没勾选任何设备时禁用按钮。
  • Kindle 邮箱和自定义设备类型不同，参数体字段也不同（mailbox vs device_url），但都在同一循环内处理。
  • 不批量推送到"邮件"——那是 /api/book/<id>/mailto 单独接口，不在本改动范围。
