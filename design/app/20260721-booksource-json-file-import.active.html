书源导入支持选择本地 JSON 文件

在书源导入对话框「粘贴 JSON 导入」页签新增按钮，允许选择本地 JSON 文件并把内容加载进已有文本框，复用既有校验与提交流程。

创建日期 · 2026-07-21模块 · app状态 · ACTIVE需求来源 · PR #883 评论

原始诉求

PR #883（书源大 JSON 导入修复）评论中，ClementLi233 提出「最好加一个选择JSON文件的按钮 可以选择复制还是加载」，liaoshutian 确认并要求「加个按钮，允许选择本地JSON文件进行上传」；随后 @claude 被要求直接实现该按钮。当前 BookSourceImportDialog.vue 的 JSON 页签只有一个 v-textarea，用户必须手动粘贴 JSON 文本，无法直接选择本地文件。

目标

  • 在 JSON 页签新增「选择本地 JSON 文件」按钮，点击后弹出系统文件选择框（限定 .json）。
  • 选中文件后读取其文本内容并填入现有 jsonText 文本框，用户可在提交前继续查看或编辑。
  • 读取失败时给出错误提示；不选择文件（用户取消）时保持原内容不变。
  • 不改变后端 API、数据结构、数据库、权限或部署配置；复用 doImport() 既有的 JSON 校验与 /admin/booksource/import 提交路径。

方案

  1. app/components/BookSourceImportDialog.vue：JSON 页签顶部新增一个 v-btn 触发按钮和一个隐藏的 <input type="file" accept=".json,application/json">；按钮点击调用 fileInput.value.click()。
  2. 文件 change 事件里用 FileReader.readAsText() 异步读取所选文件；onload 把结果写入 jsonText，onerror 通过 $alert('error', t('booksource.fileReadError')) 提示失败；读取后清空 input.value，允许重复选择同一文件。
  3. 文件内容加载进文本框后不自动提交，仍走原有的「粘贴 JSON」提交按钮与 JSON.parse 校验，行为与手动粘贴一致。
  4. 新增 i18n key booksource.loadFromFile（按钮文案）与 booksource.fileReadError（失败提示），同步补充 zh-CN.json 与 en-US.json，两者均不含字面量 @ 或 <。
  5. 组件通过 defineExpose 暴露 fileInput、onFileSelected、triggerFileSelect，供单元测试直接调用文件选择回调，无需依赖浏览器原生的文件选择对话框。

不涉及 URL 导入、内置书源导入、后端 /admin/booksource/import 接口或 webserver/main.py 中的上传大小限制（见 design/webserver/20260721-booksource-large-json-import.active.html，本方案与其相互独立、互不覆盖）。
