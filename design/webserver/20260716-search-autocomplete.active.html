搜索自动补全设计

ACTIVE 待实现 创建日期：2026-07-16 所属模块：webserver（主） + app 需求来源：features.md #38（"搜索时自动补全（模糊匹配书籍）"）

1. 原始诉求

搜索时自动补全（模糊匹配书籍）。

目前首页 / 详情页 / 网络书库等位置的搜索框仅在提交后跳转到 /search?q=... 才显示结果，输入过程中无任何提示。需要一个轻量级的"输入即提示"接口供前端自动补全组件使用。

2. 目标

  • 新增 GET /api/search/suggest?q=&limit= 接口，返回按相关性排序的书名 + 作者建议列表。
  • 前端搜索框接入 Vuetify v-autocomplete（已安装），复用现有 $backend。
  • 建议查询耗时 ≤ 200ms（万级书库用 Calibre 索引，足够）。
  • 不影响现有 /api/search 列表接口。

3. 接口

字段类型说明
qstr (query param)用户输入，最少 2 个字符
limitint (default 8, max 20)返回条数
scopestr (optional)仅搜索当前用户的可见范围（公开 / 私有）

响应：

{
  "err": "ok",
  "items": [
    {"id": 123, "title": "三体", "authors": "刘慈欣"},
    ...
  ]
}

复用现有 SearchBook 的 Calibre 查询（self.db.search_getting_ids + simple_search），限制返回字段为 id / title / authors，命中关键词优先。

4. 前端

  • app/components/SearchBox.vue（新建）：把现有 appbar 搜索框抽出来，用 v-autocomplete，绑定 /api/search/suggest。
  • 回车仍跳到 /search?q=，保证行为一致；选中提示项直接跳 /book/<id>。
  • i18n：复用 search.placeholder。

5. 边界

  • 不支持拼音 / 模糊匹配首字母；Calibre 自带的 LIKE 足够覆盖中文。
  • 不存储用户搜索历史（隐私）。
  • 不缓存建议结果（输入变化快，缓存命中率低，浪费内存）。
  • 登录前也可使用；scope 默认按当前会话 cookie 解析。
