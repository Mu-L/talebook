目标与评审结论

问题结论修复边界
私有书籍流式响应缺少 stateBookScoped 只在非流式分支调用 attach_reading_states()；前端实际使用的 ?stream=1 分支与方案声明不符。统一先格式化并批量附加状态，再由 JSON 与 NDJSON 两个分支复用。
首页测试可能弱化回归信号测试在目标书不存在时跳过核心断言，不能明确证明接口返回了测试对象。先断言目标书存在，再断言 read_state。
默认侧边栏入口分组错误默认头部把「已读书目」加入 shelf_links，位于「分类浏览」标题之前；内置主题和 ACTIVE 方案均将它归入分类区。只移动入口到分类标题之后，不改变其他导航顺序。
同页 query 导航不切换页签activeTab 只在 setup 时读取 query；从已挂载的 /user/history 点击 ?tab=finished 时页面组件可能复用，页签不会同步。监听 route.query.tab，将 finished 映射为页签 1，其余值映射为页签 0。

方案

  1. 后端在 BookScoped.get() 中生成一次 books_result，统一附加当前用户阅读状态；流式分支按结果逐条输出，避免重复格式化。
  2. 后端测试覆盖私有书籍 NDJSON 元信息和书籍行，并收紧首页目标书断言。
  3. 默认侧边栏把已读入口从书架区移入分类区；E2E 断言分类标题在入口之前。
  4. 历史页监听 query；E2E 从已挂载的历史页点击入口，验证 URL 与页签同时更新。

不改变 API 字段、数据模型、权限、角标样式、计数接口或其他主题结构。
