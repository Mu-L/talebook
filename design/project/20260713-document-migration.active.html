存量内部文档迁移方案

把内部开发资料从 document 迁入 design，并建立清晰的归属与可追溯关系

状态 · ACTIVE日期 · 2026-07-13范围 · 文档迁移
TL;DR
将 7 组日期目录中的内部方案与设计截图转为独立离线 HTML，并按主要代码归属迁入 design/<module>/。用户文档与开发者使用文档继续留在 document/。迁移不改业务实现，也不把格式检查视为历史功能测试。

01原始诉求

document/ 应只存放产品对外、面向使用者的文档；design/ 存放内部开发文档。当前已有文档需要迁移，并遵循新确定的 HTML 路径、状态和内容规范。

02目标

  • 边界清晰。读者从顶层目录即可判断文档受众。
  • 按归属检索。内部方案进入真实顶层模块；仓库级流程使用 project。
  • 历史信息不丢失。完整保留原方案中的决策、数据、风险和后续事项。
  • 结果可检查。所有迁移文件通过路径、HTML 结构、离线资源和 Git diff 检查。
