书单（BookList）功能设计

ACTIVE 待实现 创建日期：2026-07-16 所属模块：webserver（主） + app 需求来源：Issue #509

1. 原始诉求

对收藏的图书开列阅读清单，列阅读计划，或者特别开列管理。可以对现有藏书选择加入某个特别命名的阅读书单、列表。

2. 目标

  • 用户可创建多个命名的书单（私有 / 公开）。
  • 书单可加入 / 移除任意本地藏书，顺序保留，时间记录。
  • 书籍详情页和搜索结果页支持"加入书单"。
  • 用户主页新增"我的书单"入口。
  • 不与现有"书架 = 待读"语义冲突：书单是分组，书架是单状态字段。

3. 数据模型

字段类型说明
book_lists
idINTEGER PK主键
owner_idINTEGER FK→readers.id创建者
nameVARCHAR(64)书单名（owner 内唯一）
descriptionTEXT NULL可选说明
is_publicBOOLEAN是否对其他登录用户可见
created_atDATETIME创建时间
updated_atDATETIME修改时间
book_list_items（复合主键 list_id + book_id）
list_idINTEGER FK→book_lists.id书单
book_idINTEGERCalibre book id
added_byINTEGER FK→readers.id加入者（owner 或公开书单协作者）
added_atDATETIME加入时间，决定排序
noteTEXT NULL用户自定义备注（如"第 3 本必读"）

新表随启动自动迁移（compare_and_migrate 已支持新增表）。

4. API

方法路径权限说明
GET/api/lists登录当前用户的书单 + 已加入的公开书单
POST/api/lists登录创建书单（name, description?, is_public）
PUT/api/lists/<id>owner改名、改可见性
DELETE/api/lists/<id>owner删除书单（含 items 级联删除）
GET/api/lists/<id>owner 或公开书单详情 + 书籍列表
POST/api/lists/<id>/itemsowner加入书籍（body: book_id, note?）
DELETE/api/lists/<id>/items/<book_id>owner移出书籍
GET/api/book/<id>/lists登录当前书在哪些书单里（含公开）

所有接口沿用 @js + @auth 装饰器，权限校验在 handler 内做，参考 webserver/handlers/book.py:2176 shelf 的写法。

5. 前端

页面 / 组件行为
app/pages/user/lists.vue我的书单列表 + 创建按钮
app/pages/user/lists/[id].vue书单详情（书籍网格 + 备注编辑 + 删除）
app/pages/user/detail.vue侧栏加"我的书单"入口
app/pages/book/[bid]/index.vue菜单加"加入书单"按钮（弹窗多选 checkbox）
app/components/AddToListDialog.vue复用组件，传入 book_id，从 /api/lists 拉取 + 多选

6. 边界与不做的事

  • 不支持书单嵌套 / 排序拖拽（首版够用，再加）。
  • 公开书单仅允许查看，不允许其他用户编辑。
  • 不存储书单封面（用首本书的封面）。
  • 不与"书架（wants）"字段互通——避免与现有语义混淆。

8. 风险

book_id 是 Calibre 整数 id，重命名 / 迁移不会变；删除书籍时由 Calibre DB 决定是否级联。本设计不在应用层强约束孤儿 item，按 lazy 删除兜底（GET /api/lists/<id> 渲染时静默跳过找不到的书）。
