内置定制主题搜索参数修复

ACTIVE 创建日期：2026-07-21 所属模块：app 需求来源：GitHub Issue #879

1. 原始诉求

默认主题下的搜索可以搜索到书库中已有的书，而其他定制主题的搜索结果均为空白；修复该缺陷并提交 PR。

2. 根因

默认主题的 AppHeader.vue 跳转到 /search?name=...。五个内置定制主题复用 BuiltinThemeHeader.vue，该组件跳转到 /search?keyword=...。搜索页会把完整地址交给后端，而后端 SearchBook 只读取 name 参数，因此定制主题请求被识别为缺少关键字，无法返回书库结果。

3. 逐项澄清结果

决策点结论依据
接口契约统一使用现有 name 参数默认主题、后端处理器及现有 E2E 均以 name 为准
修改范围只修改内置主题共用头部五个异常主题都复用该组件；后端行为正确，无需扩大兼容面
浅灰主题分类搜索保留 title:、author:、isbn:、tag: 前缀前缀属于传入 Calibre 搜索引擎的查询内容，与 URL 参数名无冲突
空白输入保持不跳转现有交互已有防护，本缺陷不涉及变更
视觉表现不改布局与样式问题仅由路由参数不一致导致

4. 目标与方案

  • 所有内置定制主题从桌面和移动搜索入口提交后，都生成 /search?name=...。
  • 浅灰主题选择字段分类时，仍把字段前缀编码在 name 的值中。
  • 新增组件测试，直接验证普通主题与带分类的浅灰主题搜索路由，防止参数名再次漂移。
五个内置主题 → BuiltinThemeHeader.doSearch → /search?name=查询内容 → BookList → /api/search → SearchBook(name) → 书库结果

5. 影响范围

  • app/components/themes/BuiltinThemeHeader.vue：修正搜索跳转参数。
  • app/test/components/BuiltinThemeHeader.spec.ts：新增搜索路由回归测试。
  • 后端、API、数据库、配置、权限和部署均不变。
