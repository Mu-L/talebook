对齐 Slim Base 的 CI 环境探测

创建日期：2026-07-17 所属模块：project 状态：ACTIVE 需求来源：PR #873 test-server 失败
test-server 不再探测 slim base 已移除的 GUI 入口 calibre，改为验证测试真正依赖的 ebook-convert、calibredb、make、Python 与 pip。镜像内容和测试执行链保持不变。

原始诉求与失败证据

用户确认新版 base 不提供 calibre --version，要求同步更新 GitHub Actions workflow。PR #873 的 run 29590332166 已成功拉取新镜像并通过 actions/checkout 的 REST fallback 下载源码，随后在 Inspect environment 的首条 which calibre 以退出码 1 终止，尚未执行依赖安装和后端测试。

观察设计结论
新 base 只公开服务端所需 Calibre CLI环境探测应检查 ebook-convert 与 calibredb，不应以 GUI 聚合入口作为验收条件。
后续步骤使用 make init、make lint-py、make pytest同时检查 make，使失败在环境探测阶段清晰暴露。
checkout 已无 Git 成功不引入 Git，也不恢复 safe-directory 命令。

方案与边界

文件计划改动
.github/workflows/ci.yml将环境探测更新为 command -v 和版本检查：ebook-convert、calibredb、make、python3、pip3。
tests/test_ci_workflow.py回归固定必需命令，明确禁止再次检查 calibre GUI 入口。

不修改 base 镜像，不增加 Git 或 GUI 组件，不改变 test-server 直接运行于 base container 的方式，也不在 runner 内二次构建 Docker test target。

测试结果

本地验证完成：回归测试、发布镜像命令探测以及 slim base 内完整后端 CI 均通过；PR #873 的远端 test-server 与 integration checks 在推送后继续确认。

风险与回滚

  • 版本命令可能初始化 Calibre 运行环境；以 base 发布时已通过的 CLI smoke test 为依据，并在发布镜像上直接复验。
  • 若未来测试依赖新增命令，应按实际调用更新环境契约，而不是恢复未被业务使用的聚合入口。
  • 回滚仅需恢复 workflow 探测列表，不影响镜像和业务代码。
