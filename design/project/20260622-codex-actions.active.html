Codex GitHub Actions 配置

让维护者从 issue 与 PR 触发受限的 Codex 任务，同时隔离未信任代码、登录态与写权限

状态 · ACTIVE原日期 · 2026-06-22迁移 · 2026-07-13
TL;DR
.github/workflows/codex.yml 接受维护者通过 @codex 或 /codex 发起任务。workflow 只 checkout 可信 master，将 PR diff 作为文本交给 Codex；Codex 无网络、无 checkout 写 token、不会自动 push，产生的修改以 artifact 交付。

01原始诉求

需要让仓库维护者在 issue 评论、PR 评论、PR review 和 review thread 中直接触发 Codex 协助分析或修改代码，同时避免 privileged comment workflow 执行未信任 PR 代码或泄露 ChatGPT 登录态。

02目标

  • 只允许具有 write、maintain 或 admin 权限的评论者触发。
  • PR 上下文可被分析，但 PR head 与 merge ref 不进入带 secret 的执行环境。
  • Codex 只能在 workspace-write 沙箱中修改工作区，agent 阶段关闭网络。
  • 默认不 push；变更作为 artifact 输出，由维护者审阅。

03方案

Secret 配置

仓库 secret CODEX_AUTH_JSON 保存本地 ~/.codex/auth.json 的完整内容。若登录态默认存于系统 keychain，先在 ~/.codex/config.toml 配置文件凭据存储，再重新登录。

cli_auth_credentials_store = "file"

运行时边界

控制点约束
认证文件写入 runner 临时目录，权限 600，不进入仓库工作区。
安装顺序先安装 Codex CLI，再恢复认证文件，避免依赖安装接触登录态。
代码来源始终 checkout 可信 master；PR files 与 diff 通过 API 读取。
Agentworkspace-write 沙箱，运行阶段无网络。
Git 权限checkout 不提供写 token，默认仅能读代码并写 issue/PR 评论。
环境变量排除常见 token、secret、key，避免子进程继承。
输出不自动 push；有 diff 时上传 artifact 并提示维护者。
长期预留
若未来允许直接向 PR 分支推送，必须新增细粒度 CODEX_GITHUB_TOKEN，并把推送放入独立 job；不能扩大现有 agent job 的默认权限。
