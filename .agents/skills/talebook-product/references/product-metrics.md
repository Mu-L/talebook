# Talebook 产品指标

## 选择原则

1. 每个方案选择一个主要结果指标和少量护栏指标。
2. 优先使用产品能够真实采集的事件或服务端状态；没有埋点时先定义采集方案。
3. 不编造当前基线、用户数或目标提升比例。
4. 同时观察读者价值、管理员成本、可靠性和安全护栏。
5. 为指标写清口径、时间窗口、分母、排除条件和数据来源。

## 指标候选

| 产品环节 | 结果指标 | 护栏或诊断指标 |
| --- | --- | --- |
| 首次部署 | 从打开安装页到成功进入书库的时间与完成率 | 自检失败类型、回退次数、配置错误数 |
| 入库与扫描 | 成功入库率、首次成功入库时间 | 重复率、失败可恢复率、人工干预次数、任务耗时 |
| 元数据整理 | 一次确认后可接受的元数据比例 | 来源失败率、字段回退率、人工修改字段数、误覆盖率 |
| 搜索与发现 | 搜索或筛选后打开目标书籍的完成率 | 无结果率、改写查询率、响应时间、筛选撤销率 |
| 阅读 | 打开书籍到可阅读内容出现的成功率与时间 | 转换失败率、加载超时、异常退出、格式差异 |
| 跨端连续性 | 在另一设备成功继续到预期位置的比例 | 同步延迟、冲突率、回退位置、设备识别失败 |
| 笔记与高亮 | 创建后可再次读取的成功率 | 同步失败、定位漂移、重复记录和丢失率 |
| 设备推送 | 目标设备成功收到可打开文件的比例 | 转换失败、邮件/上传失败、重试次数和耗时 |
| 管理维护 | 完成升级、诊断或恢复任务的时间与成功率 | 回滚率、日志可定位率、配置漂移和支持请求量 |

## 输出口径

为每个选定指标给出：

- 指标名称与用户意义。
- 计算公式和数据来源。
- 观察窗口与分群方式。
- 当前是否可采集；若不可采集，列最小埋点或日志改动。
- 可能被优化“钻空子”的方式和对应护栏。
