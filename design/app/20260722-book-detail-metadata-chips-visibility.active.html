01. 原始诉求与缺陷表现

用户提供默认主题与水墨主题的书籍详情对比截图，并指出前端标签没有显示。默认主题中标签区域只看得到“公共书籍”；水墨主题中作者、出版社、普通标签及“公共书籍”均正常可见。

经确认，DOM 与接口数据都存在，缺陷属于颜色规则冲突造成的视觉不可见，不是后端漏字段，也不是 Vue 条件渲染失败。

现场观察判断
默认浅色主题作者、出版社、普通标签不可见；绿色“公共书籍”可见tonal 标签受错误文字色影响，flat 状态标签仍有实体背景
水墨主题同一组元数据标签全部可见内置主题覆盖层重新指定 tonal 配色，暂时掩盖页面级错误规则
接口与模板authors、publisher、tags 均有值且对应 Chip 已渲染无需修改 API、字段或模板数据流

03. 目标与边界

必须达成

所有主题中清晰、完整、可用

  • 默认主题以及五个内置主题的浅色、深色模式均可清楚辨认元数据 Chip。
  • 作者、出版社、系列、ISBN、普通标签全部覆盖；有链接的 Chip 保持可点击。
  • 公有、私有状态继续保留 teal / orange 语义色。
  • 缺少可选元数据时继续遵循现有条件渲染，不制造空标签。

明确不做

不扩张数据与主题契约

  • 不修改后端 API、数据库、权限或 Calibre 元数据。
  • 不改变 Chip 的文案、顺序、跳转地址、尺寸或间距。
  • 不为单个主题硬编码新的颜色。
  • 不重构整页样式或内置主题系统。

04. 实现方案

06. 风险与回滚

风险控制

  • 移除强制白字可能影响状态 Chip：通过公有与私有两种状态分别验收。
  • 单纯 DOM 可见不代表视觉可见：E2E 同时检查默认浅色的计算样式，浏览器检查全部主题矩阵。
  • 内置主题覆盖可能隐藏新冲突：浅色和深色必须分别实际渲染。

回滚策略

改动限定为一条页面级 CSS 与测试。如果产生主题回归，可整体回退修复提交；不涉及数据、API 或迁移，不需要额外回滚步骤。
