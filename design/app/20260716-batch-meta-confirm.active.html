批量更新元数据确认页设计

ACTIVE 待实现 创建日期：2026-07-16 所属模块：app 需求来源：features.md #67（"批量更新元数据（需要有个表格确认页？）"）

1. 原始诉求

批量更新元数据（需要有个表格确认页）。

当前 AutoFillService.auto_fill_all 一旦触发就会按数据源直接覆盖（replace_metadata=True），管理员无机会逐本预览差异。一旦豆瓣 API 返回错误元数据，整批书籍都会被污染。需要在前端加一道"预览 → 确认"的闸门。

2. 目标

  • 管理员在 /admin/books 勾选多本书 → 点"批量更新元数据"→ 弹出确认页（不是直接执行）。
  • 确认页显示每本书的新旧字段对比，管理员可逐本勾选"应用 / 跳过"。
  • 可选择"仅更新空字段"或"全量覆盖"。
  • 提交后异步执行，进度轮询。

3. 后端最小改动

现有 /api/admin/book/fill（webserver/handlers/admin.py）直接执行。本设计在它之前加一个"预览"接口：

方法路径说明
POST/api/admin/book/fill/preview接受 book_id 列表，返回每本的新旧元数据 diff（不落库）
POST/api/admin/book/fill现有接口保持不变；新增可选参数 preview_id（引用某次预览结果），按勾选项执行

预览接口复用现有 AutoFillService.plugin_search_best_book_info，不调用 smart_update；只计算 diff 返回。

4. 前端确认页

位置内容
app/pages/admin/books.vue多选后按钮文案改为"批量更新元数据（预览）"
app/components/BatchMetaConfirmDialog.vue（新建）表格：书名 / 作者 / 出版社 / 简介 / 标签 / 封面（缩略图），每行一对新旧值，高亮差异
字段控制每列顶部 checkbox："更新该字段"，默认全勾
顶部策略切换"仅更新空字段"（推荐）/ "全量覆盖"（管理员自负责）
提交调 /api/admin/book/fill，传入勾选项 + 策略；进度用现有轮询接口

5. diff 展示

对每个字段，返回 {old: ..., new: ...}；前端用 Vuetify 表格展示：

  • 相同字段：灰底 + 折叠按钮展开对比。
  • 不同字段：黄底标记，未勾选则提交时跳过。

6. 边界

  • 预览请求可能耗时（每本都要调外部 API）。一次预览最多 50 本，超出分批。
  • 单本预览失败时该行标红，仍可继续其他书籍。
  • 预览结果不持久化（纯内存）；关闭 dialog 即失效。
