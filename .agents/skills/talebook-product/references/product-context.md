# Talebook 稳定产品上下文

本文件只保留较稳定的产品事实和决策边界。日期相关证据放入独立快照；路线图和重大研究开始前必须重新采集当前证据。

## 定位

Talebook 是基于 Calibre 的自托管个人电子书管理系统，强调简单部署、书库管理、在线阅读、多用户与小圈子共享。生产形态由 Nginx 承载 Nuxt 前端并代理 Tornado 后端，主要面向 NAS、家庭服务器和个人服务器环境。

关键约束：

- 私有、自托管优先，不把公开出版或公共书库运营作为默认场景。
- 同时服务实例管理员和读者；管理员维护成本也是产品成本。
- 通过 Web、Moke、OPDS、WebDAV 和设备推送连接外部阅读生态。
- 依赖 Calibre 的格式转换、元数据数据库和文件组织能力，必须考虑升级与多架构成本。

## 角色与核心任务

| 角色 | 核心任务 | 常见风险 |
| --- | --- | --- |
| 实例管理员 | 安装升级、配置、导入扫描、元数据、权限、安全、备份和恢复 | 配置复杂、长任务不可见、升级不兼容、安全边界、数据损失 |
| 个人读者 | 搜索发现、组织、阅读、进度、笔记、推送和跨端继续 | 状态不一致、转换等待、进度丢失、移动端摩擦 |
| 家庭/小圈子成员 | 注册登录、浏览共享内容、维护个人记录 | 公私范围混淆、权限反馈不清、个人数据泄露 |
| 访客/外部客户端 | 受限浏览、下载和协议接入 | 认证复杂、兼容差异、错误难诊断 |

## 产品闭环

1. **部署维护**：Docker 安装、初始化、配置、更新、自检、日志、安全与备份。
2. **入库清洗**：上传、扫描、外部导入、去重、回收站和格式转换。
3. **元数据整理**：多源检索、AI 识别、批量更新、封面、系列、标签和作者。
4. **发现组织**：首页、搜索、筛选、书架、阅读状态、书单和私藏范围。
5. **阅读同步**：多格式阅读、转换等待、进度、笔记、高亮和桌面端。
6. **分享设备**：多用户、访客权限、设备推送、OPDS 和 WebDAV。

## 产品原则

1. 优先保证核心闭环的一致性、可恢复性和可解释性，再扩展内容品类。
2. 将安装、升级、扫描、诊断和备份体验视为产品体验，不把它们降格为纯运维问题。
3. 跨端能力以清晰契约为基础：区分 Web、Moke、阅读器、OPDS 和 WebDAV 能表达的状态。
4. 批量和长任务必须考虑阶段、进度、失败项、重试、审计和后台继续。
5. 中文元数据与书源生态可以形成差异化，但不得牺牲 SSRF、脚本、登录态、付费内容和反爬边界。

## 外部边界

- candle-reader：Web 阅读器能力与阅读状态契约。
- Moke：桌面端浏览、下载、离线阅读和认证体验。
- Calibre：格式转换、元数据数据库、文件组织和工具链。
- 外部客户端：仅承诺 OPDS、WebDAV 或其他协议实际能表达的能力。

涉及上述边界时检查对应仓库或官方协议资料，不根据 Talebook README 推断其最新实现。
