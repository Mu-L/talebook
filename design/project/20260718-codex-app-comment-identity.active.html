03 · 实现方案

位置变更不变量
job permissions把默认 GITHUB_TOKEN 收敛为仅 contents: read，使漏传 App Token 的写操作 fail closed，而不是回退成 github-actions[bot]。actions/checkout 仍可读取仓库；不持久化凭据。
job trigger将仅排除 github-actions[bot] 改为排除所有 [bot] actor，防止专用 App 回复正文含触发词时递归。仍要求事件作者关联为 OWNER、MEMBER 或 COLLABORATOR，并在脚本内复核 write 级权限。
interaction_token在 context 前通过现有 App Secrets 创建低权限 Token；用于所有 actions/github-script 交互步骤和 progress reporter。不提供 contents: write；Token 不进入 Codex 子进程。
context / unsupported为 Resolve request context 与 Explain unsupported target 显式设置 github-token: interaction_token。权限检查、目标解析、受信任资源 SHA、中文失败路径保持原逻辑。
实时进度CODEX_PROGRESS_TOKEN 改为 interaction_token；Python reporter 继续只持有评论 ID 与短期 Token。env -u CODEX_PROGRESS_TOKEN codex exec 保持，避免 agent 读取凭据。
最终答复为 Post Codex response 显式传入 interaction_token，继续更新同一条由 App 创建的进度评论。结构化结果、发布状态、补丁 artifact 和 64K 截断保持不变。
publisher app_token保留 publish gate 后的 Token 创建条件与现有发布/PR 用法，仅调整测试名称以明确职责。未通过门禁时不产生 contents: write Token；禁止 force push。

04 · 安全、失败处理与兼容性

  • 凭据隔离：交互 Token 只传给受信任的 GitHub Script 和默认分支版本固定的 reporter；Codex 进程继续通过 env -u 删除该变量，shell 环境排除规则继续覆盖所有 Token。
  • 失败关闭：交互 Token 创建失败时，不允许用默认 GITHUB_TOKEN 降级发言；job 在任何 App 可见写操作前失败，Actions 日志给出凭据错误。
  • 发布失败可见：publisher Token 或 push 失败时，独立的 interaction Token 仍能以 App 身份写入最终失败说明与恢复补丁信息。
  • 已有评论：历史评论作者不可重写；新 workflow 仅影响新触发执行。更新旧评论不作为本 PR 的迁移目标。
  • App 权限：复用当前已安装 App；若安装范围移除仓库或权限不足，Token action 失败并按失败关闭处理，不回退到个人 PAT。
  • 运行时：25 分钟 job 与 20 分钟 Codex 预算不变；两个短期 Token 均在同一有界 job 内使用。
发布与回滚本 PR 只调整 workflow 凭据路由、契约测试和方案生命周期，不改变 Codex Prompt、模型、sandbox 或发布分支策略。若上线后 App 评论权限异常，通过新替代方案恢复原 Token 路由；ACTIVE 方案不得直接删除或回写核心决策。
