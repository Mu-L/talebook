接入 txt2epub TXT 转 EPUB 转换器

创建日期2026-07-23 所属模块webserver（影响 app、Docker 构建与 Python 依赖） 状态ACTIVE 需求来源用户请求：将 hehetoshang/txt2epub 加入 Talebook 的格式转换模块，可作为子模块使用。

原始诉求与目标

使用 txt2epub 为 Talebook 的 TXT 转 EPUB 场景生成更适合小说章节识别、编码识别和目录组织的 EPUB。将书籍页分散的格式转换入口合并为“转换书籍”，在弹窗中展示当前格式、可用路径与不可转换原因；转换请求必须明确提交源格式和目标格式，避免后端猜测错误。

设计决策与边界

问题结论原因
接入范围当前只开放 TXT 输入且 EPUB 输出的转换能力。txt2epub 的职责就是 TXT 转 EPUB；保留旧 Calibre 服务能力供内部兼容，但不在新弹窗中暴露未确认的转换路径。
源码管理以 vendor/txt2epub Git 子模块固定上游提交。保留上游历史与可追溯版本，避免复制源码。
调用方式由后端加载子模块 Python API，而非依赖全局命令行安装。容器和本地运行均能使用仓库内固定版本；直接传递 Talebook 的标题与作者，随后沿用既有 Calibre 元数据写回（包括封面）。
失败处理转换失败返回 false，沿用既有日志、任务完成和用户失败通知。不改变异步任务的对外语义。
能力判定后端返回每条转换路径的 available 与 reason，前端只负责展示。防止前后端支持范围漂移；缺少 TXT、已有 EPUB、路径不支持均由后端再次校验。
交互入口书籍处理菜单只保留一个“转换书籍”入口，点击后打开弹窗。弹窗展示当前格式、TXT → EPUB 路径、工具说明和禁用原因，确认按钮只在路径可用时启用。
API 参数POST /api/book/:id/convert 接收 source_format 与 target_format。消除旧实现按固定顺序挑选源文件导致 txt2epub 未被调用的问题。
容器加载生产镜像复制 vendor/，开发 Compose 显式挂载 vendor/txt2epub；后端按子模块绝对路径加载包。修复开发容器只挂载 webserver/ 时出现的 No module named 'vendor'，并在子模块缺失时给出可执行的错误提示。

实现流程

验证计划

  • 为 TXT→EPUB 分支添加服务单元测试：传递元数据与封面，验证成功和异常路径。
  • 为转换能力判定和显式 API 参数添加后端用例：可用、缺少源格式、目标已存在和不支持路径。
  • 为转换弹窗添加组件测试：可用状态、禁用原因和确认事件。
  • 启动 mock API 与 Nuxt，在书籍详情页验证处理菜单、弹窗、可用及禁用状态。
  • 验证非 TXT→EPUB 仍调用 Calibre。
  • 运行相关 pytest、后端 lint、设计文档检查；如环境允许，运行完整后端测试。
  • 确认 Docker 镜像复制子模块并安装 txt2epub 所需第三方依赖。

本地 pytest未执行：当前默认 Python 缺少 pytest 与项目运行依赖；尝试按 requirements 安装时，Python 3.14 与固定的 tornado 6.5 不兼容。使用项目随附的 Python 3.12 安装临时依赖时，配置的 PyPI 镜像在 120 秒内无响应。
Docker 实测未执行：Docker 配置与 daemon 管道均被沙箱拒绝；权限审批服务又发生模型配置错误。镜像三个阶段均已复制 vendor/，开发 Compose 已补充只读挂载。建议在有 Docker 权限的环境运行 docker build --target test -t talebook/txt2epub-test .，再运行 docker run --rm talebook/txt2epub-test pytest /var/www/talebook/tests/test_service.py::TestConvert -v。
运行时回归用户实测先后发现顶层 vendor 不在 Python 搜索路径，以及失败导入留下半初始化模块。现改为按绝对路径加载，失败时清理包及子模块缓存并允许重试；缺少第三方依赖时提示重建 Docker 镜像。已新增半初始化缓存恢复用例。
浏览器验收未执行：浏览器安全策略拒绝访问本地 127.0.0.1 页面。交互已由组件测试与 mock API 场景覆盖，但仍建议在开发机复核书籍详情页的可用和禁用弹窗。

剩余风险：当前受沙箱与本机 Python 依赖限制，真实 EPUB 生成、Docker 回归和浏览器渲染未在本次会话中执行。实现同时提供了真实转换测试、接口测试、组件测试和显式缺失提示；后续应在项目 Docker 环境补跑上述命令。
