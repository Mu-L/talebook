01 · 原始诉求与验收目标

PR #870 的评审指出三个 P2：本地监听被禁止导致无法完成前端验收；无 diff 路径可跳过仓库内残留 WIP 检查；最终评论没有可靠展示结构化测试命令和结果。

目标验收标准
本地服务可用permission profile 允许 agent 在 loopback 上监听，并能访问 localhost、127.0.0.1。
无变化也 fail closed分支存在 *.wip.html 时，即使没有新 diff，也不得成功或补建 Draft PR。
测试证据可审计结构化契约有效时，评论必须展示其 summary、测试命令、结果和可选详情。

02 · 决策与安全边界

已确认的取舍allow_local_binding = true 同时允许 loopback 监听并放宽网络代理的本地/私网保护。维护者确认接受该取舍，以满足仓库强制的前端实际渲染验证。
  • Agent 仍运行在一次性 GitHub-hosted Runner，文件系统默认拒绝读取工作区外内容。
  • CODEX_AUTH_JSON、GitHub token、App 私钥和常见 Secret 环境变量继续对 agent shell 隐藏。
  • 不开放任何 Unix socket；Prompt 继续禁止访问私网，并在域名策略中显式拒绝常见云元数据地址。
  • .github/workflows 保持 read-only，发布 token 仍只在 Codex 结束后创建。

03 · 门禁和评论数据流

评论规则

  • 仅当 publish gate 输出 contract_valid=true 时解析结构化文件。
  • summary 作为评论正文事实摘要，tests 渲染为独立 Validation 列表。
  • 契约无效或执行失败时才退回 Codex 最终自然语言或通用错误提示。
  • 提交 SHA、分支、Draft PR、失败原因和恢复 artifact 仍由 workflow 追加，不能被 agent 覆盖。
