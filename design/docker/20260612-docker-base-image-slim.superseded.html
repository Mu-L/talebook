在不损失 Calibre 转换能力的前提下，去掉安装方式造成的重复与隐式依赖

TL;DR
静态化 ebook-convert 不可行：转换链依赖 Python、Qt6 Gui 与 QtWebEngine，服务端也直接导入 Calibre。第一阶段清理 apt Recommends 与重复 Qt5，同时显式保留编译、图片格式和中文字体依赖。2026-07-13 核对确认瘦身改动仍在 Dockerfile.base 中，但主镜像仍使用 base 8.5，尚未切换到计划发布的 8.6。

01原始诉求

Talebook arm64 完整镜像实测 2.63GB，需要评估能否把 ebook-convert 编译成静态工具并缩小镜像，同时不能牺牲现有格式转换和 PDF 输出能力。

02目标

  • 第一阶段以零功能损失为硬约束。
  • 移除与 Debian 13 / Calibre Qt6 运行时重复或无关的包。
  • 把过去由 Recommends 隐式带入、实际仍必需的依赖显式化。
  • 先发布 base 8.6，再切换主 Dockerfile，避免 CI 依赖尚不存在的标签。

03方案与结果

静态化结论
ebook-convert 是 Python 程序；封面与图片依赖 Qt6 Gui，PDF 输出依赖 QtWebEngine 的无头 Chromium。Calibre 官方二进制同样为 GB 级，且 webserver 有 17 处直接 import calibre，因此不能在本阶段解耦为小型静态工具。

依赖调整

  1. 全程使用 --no-install-recommends。避免拉入 scipy、matplotlib、GPU 驱动、gcc、vim 等无关包。
  2. 移除 python3-pyqt5*。Debian 13 的 Calibre 已使用 Qt6 / PyQt6，旧 Qt5 与 Chromium 属于重复安装；验证逻辑同步改为 PyQt6。
  3. 显式保留构建工具。build-essential 与 python3-dev 用于构建无预编译 wheel 的依赖，例如 quickjs。
  4. 显式保留 Qt6 图片插件。qt6-image-formats-plugins 与 qt6-svg-plugins 保证 WEBP、TIFF、SVG 内容与封面支持。
  5. 补装中文字体。fonts-wqy-microhei 与 PDF 转换服务的硬编码字体一致。
  6. 多架构发布。push-base 使用 buildx 发布 amd64、arm64 与 arm/v7。

发布顺序

主 Dockerfile 在第一阶段继续使用 talebook/talebook-base:8.5。先推送 base-v8.6.0 触发多架构构建，确认 :8.6 可拉取后，再合入第二阶段的主镜像切换与 slim 变体。
