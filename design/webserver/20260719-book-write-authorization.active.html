书籍写接口所有权校验修复

ACTIVE 创建日期：2026-07-19 所属模块：webserver 需求来源：PR #871 维护者评审请求

原始诉求

评审并修复发现的问题，推送到当前分支里。

PR 修复了 BookEdit、BookDelete、BookSetScope、BookDeleteFormat 和 BookSeparate 的所有权校验绕过，并为 BookSetScope 新建的 Item 设置 collector_id。评审发现该安全修复缺少测试与方案记录，无法证明所有入口均已封堵，也无法防止后续回归。

目标与边界决策

  • 五个写接口必须使用 self.user_id() 校验当前登录用户，而不是使用书籍响应中的收藏者 ID 自证所有权。
  • 管理员仍可绕过所有权限制；can_edit 与 can_delete 权限位的既有语义不变。
  • 保持现有路由、请求体和错误响应不变，越权请求继续返回 {"err": "permission"}。
  • 当管理员为没有 Item 的书籍切换范围时，新记录的 collector_id 必须是当前操作者，不能依赖模型默认值。
  • 不扩展到本 PR 未修改的只读接口或管理员批量接口。
