已读标记角标方案

在书籍封面网格与列表上直接展示「已读」角标，并在侧边栏加入「已读书目」快捷入口

状态 · WIP关联 · issue #853 / issue #350创建 · 2026-07-16
TL;DR
后端已有完整的 ReadingState.read_state（0 未读 / 1 在读 / 2 已读）能力，但书籍列表类接口（首页推荐、最近上架、书库/分类浏览等）此前不透出该字段，前端封面也没有任何视觉标记。本次给 BaseHandler 新增 attach_reading_states()，让已登录用户在浏览任意书籍网格时都能看到统一的角标（封面右上角圆形对勾）与列表内联「已读」标签；同时在侧边栏「分类浏览」下新增「已读书目」入口，跳转到既有的 /user/history?tab=finished 已读完页签。

01原始诉求

issue #853 引用 issue #350 的截图：某第三方书库应用在首页「随机书籍」封面右上角展示蓝色对勾角标标记已读书籍，「最新书籍」列表标题旁展示「已读」小标签，侧边栏「分类浏览」下有「已读书目」快捷入口并显示数量。issue 标题「将会将已读标记改为这种」表达的诉求是：把 talebook 现有「已读」状态在浏览页面中做出等价的可视化标记。

02目标

  • 已登录用户浏览首页推荐、最近上架、书库/分类页、私有书籍等封面网格时，凡是自己标记为「已读」的书籍，封面上出现统一的角标。
  • 在以标题为主的列表卡片（BookCards/BookCards_Small）中，标题旁追加「已读」小标签，与已有 readingState.done 文案保持一致。
  • 侧边栏新增「已读书目」入口（仅登录用户可见），点击后进入已有的「已读完」页签，并显示当前已读数量。
  • 未登录或未标记的书籍不受影响，不新增任何数据模型，复用既有 ReadingState。

03方案

层级实现
后端·新增方法BaseHandler.attach_reading_states(formatted_books)：已登录时按书籍 id 批量查询 ReadingState，为每本书写回完整的 state（含 read_state）；未登录时统一写回 read_state=0，字段始终存在。
后端·接入点ListHandler.render_book_list() / stream_book_list()（覆盖 /api/recent、/api/library、/api/hot、/api/book/nav 等所有基于 ListHandler 的路由）、Index.get()（/api/index 首页推荐与最新）、BookScoped.get()（/api/scopedbooks 私有书籍）。
前端·新增组件components/BookReadBadge.vue：接受 readState，仅当值为 2 时渲染封面右上角蓝底白色对勾圆形角标。
前端·接入点pages/index.vue 首页随机推荐封面、components/BookCards.vue、components/BookCards_Small.vue：封面叠加角标；后两者额外在标题旁追加「已读」v-chip（复用既有 readingState.done 文案）。
前端·侧边栏AppHeader.vue 与 themes/BuiltinThemeHeader.vue 在「分类浏览」下新增「已读书目」入口（仅 store.user.is_login 时展示），链接 /user/history?tab=finished，挂载时调用既有 /api/read-done 获取 total 作为角标数字。
前端·已读完页签pages/user/history.vue 读取 route.query.tab === 'finished'，据此默认选中「已读完」页签，使侧边栏入口可以直达。
范围边界
本次只做「浏览态角标展示」，不改变 ReadingState 数据模型、不改变已有的收藏/书架/在读/已读完专用列表接口的返回结构（它们本就自带完整 state）。network/index.vue 的外部搜索结果、BookCoverGrid.vue（已读完页签自身）未接入角标：前者书籍不对应本地 ReadingState，后者语境已经是「已读完」列表，角标冗余。若后续需要侧边栏按截图顺序把「已读书目」置于分类列表最前（当前排在「分类浏览」标题之后、其余分类之前，不改变其余分类顺序），需要产品确认后再单独调整导航排序。
