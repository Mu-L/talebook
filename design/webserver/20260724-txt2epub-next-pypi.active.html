原始诉求与目标

Talebook 当前以 vendor/txt2epub Git 子模块加载转换器。迁移为安装已发布的 txt2epub-next PyPI 分发包，删除子模块和其容器挂载或复制逻辑，同时保持 TXT 转 EPUB 的现有调用契约和转换行为。

方案

以发布包作为唯一来源。依赖清单显式声明包名和已验证版本；服务层使用发布包提供的稳定 import 路径直接导入 Txt2Epub，移除按文件路径动态加载、partial-module 恢复和“子模块缺失”错误。Docker 测试、生产与开发镜像不再复制 vendor/，开发 Compose 不再挂载该目录，Git 元数据删除对应子模块记录。

决策实施边界
分发来源在 requirements.txt 与项目元数据中记录 txt2epub-next，由 pip 安装。不保留本地源码或 Git URL 回退，构建可复现性以版本约束保证。
导入契约根据已发布 wheel 的顶层包名，直接导入 Txt2Epub。若分发名与 import 名不同，只在此处适配，不改变转换 API 和前端的 converter: txt2epub 识别值。
清理范围移除 .gitmodules、Gitlink、Dockerfile 的 vendor 复制及 docker-compose.dev.yml 的挂载。不删除无关的 vendor/ 目录，除非该目录迁移后为空且只含此子模块。
