自动更新图书标签修复

ACTIVE 待实现 创建日期：2026-07-16 所属模块：webserver 需求来源：Issue #522（"希望自动更新图书信息可以自动更新图书标签"）

1. 原始诉求

希望【图书管理-自动更新图书信息】功能可以自动更新图书标签。

管理员后台的批量自动填充任务（AutoFillService.auto_fill_all）以及单本刷新（/api/book/<id>/refer）执行后，用户期望标签会像封面 / 简介一样被一并更新。实际行为存在两处断裂：

  1. 豆瓣 V2 插件返回的元数据完全没写 mi.tags。
  2. autofill.py 定义了 guess_tags()，但 do_fill_metadata() 走完 smart_update 后从未调用它。

2. 目标

  • 任何开启了"自动更新元数据"的数据源（含豆瓣 V2、AI 等）刷新成功后，标签字段都被写入。
  • 保留：豆瓣 V1 已经会写标签，不要回归。
  • 不动：标签的手动编辑、导入时从 EPUB 本身抓取的逻辑。

3. 现状定位

位置现状
webserver/services/autofill.py:96-128 do_fill_metadataL125 调 mi.smart_update(refer_mi, replace_metadata=True) 后立即落库，未调用 guess_tags。
webserver/services/autofill.py:135 guess_tags定义完整，按 BOOK_NAV 字符串匹配书名 / 简介 / 作者，最多 8 个。
webserver/plugins/meta/douban_v2/api.py:234 build_metadata完全没提取 tags 字段。豆瓣 V1 在 L205 写 mi.tags = [t["name"] for t in book["tags"]][:8]。
webserver/handlers/book.py:218 BookRefer.post手动刷新走的是 smart_update，所以豆瓣 V1 刷新能带标签、V2 不能。

4. 方案

4.1 豆瓣 V2 提取标签

在 build_metadata 中按 V1 的语义补上：

tags = [{"name": t.get("name")} for t in item.get("tags", []) if t.get("name")]
if tags:
    mi.tags = [t["name"] for t in tags][:8]
    mi.tag_sort = mi.tags

若 V2 接口实际不返回 tags，调用点退化为空列表（保持现状即可，不报错）。

4.2 复活 guess_tags

在 do_fill_metadata 中 smart_update 之后、落库之前插入：

if not mi.tags:
    guessed = self.guess_tags(refer_mi)
    if guessed:
        mi.tags = guessed

guess_tags 仅在标签为空时触发，避免覆盖已经存在的真实标签。

5. 边界

  • smart_update(replace_metadata=True) 默认会用 refer 的 tags 覆盖原有标签；若 refer 无 tags（如 AI、Calibre、NeoDB），会清空原标签。因此新加的 guess_tags 兜底必须放在 smart_update 之后。
  • 已存在的真实标签（如用户手动加的）由 smart_update 保留；本次不调整该语义。
