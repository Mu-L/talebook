PROJECT GUARDRAIL / FILE WATCHING

禁止启用 Chokidar 轮询模式

把一次真实的 CPU 异常转化为仓库级硬约束：任何智能体都不得在 Talebook 的开发命令中加入 Chokidar polling 开关。

ACTIVE · 已生效 创建日期 2026-07-22 模块 project 需求来源 用户现场指令

原始诉求与原因

用户明确要求：“备注到 AGENTS.md 里面，永远不要加 CHOKIDAR_USEPOLLING=true。”

本次实际启动 Nuxt dev server 时加入该环境变量，Chokidar 因此持续轮询前端目录，造成显著 CPU 占用。本机工作区支持原生文件事件，不需要以全目录轮询换取热更新。

规则的唯一解释

CHOKIDAR_USEPOLLING=true npx nuxt dev
×
npx nuxt dev --port 3000 --host 127.0.0.1
  • 这是绝对禁令，不为 Docker、worktree、网络文件系统或热更新故障预设例外。
  • 禁止在交互命令、脚本、Makefile、CI、文档示例或临时诊断命令中设置 CHOKIDAR_USEPOLLING=true。
  • 文件监听异常时，先停止 dev server 并报告现象；使用原生文件事件、手动刷新或其他不启用 Chokidar polling 的方案。
  • 启动后出现异常 CPU 占用时，立即停止由智能体启动的进程并检查残留，不继续带病运行。

落地位置

AGENTS.md

在“前端验收”本地启动说明附近增加醒目的禁止项，让智能体在复制启动命令时直接看到。

现有启动示例

保留当前不含 polling 的标准命令；不修改应用代码、依赖、Nuxt 配置或 Docker 配置。

风险与回滚

风险仅在于某些特殊文件系统的热更新可能失效。按照用户给出的绝对约束，该情况下也不启用 polling，而是明确报告并选择手动刷新或其他低 CPU 方案。规则如需改变，必须新建工程治理 WIP 方案，不能静默删除或弱化。
