原始诉求与核对结果

原始任务是处理 PR #869 中评审发现的问题。该 PR 的基础方案为 design/webserver/20260716-admin-private-book-access.active.html ，本方案是兼容性补充，不替代其管理员权限矩阵。

评审项核对结论处理决定
can_view_book() 与既有 can_read_book() 重复当前 master、PR HEAD 与本地所有引用中都不存在 can_read_book()，无法复现重复。保留唯一的 can_view_book()，不凭空新增第二套授权函数；在 PR 回复中说明证据。
阅读入口的旧检查成为死代码当前代码没有这些旧调用；但中央 get_book() 会让私藏拒绝提前以 HTTP 200 JSON 结束。非 JSON 文件与阅读入口统一调用 get_book_or_404()，由该助手复用中央授权并在不可见或不存在时返回 404。
配置 static_host 后私藏封面 404成立。序列化仍把封面、缩略图和格式下载 URL 指向 CDN 域名，而认证 Cookie 属于应用域名。私藏资源使用应用源；公开资源继续使用 CDN。

目标与不变量

  • 管理员和所有者在配置 static_host 时仍能加载私藏封面、缩略图和下载链接。
  • 公开图书继续使用 static_host，不牺牲现有 CDN 缓存能力。
  • 无权用户访问 /read/<id> 或 /get/extract/<id>/... 时返回 HTTP 404，不泄露图书存在性。
  • 权限矩阵保持不变：管理员可见全部，普通用户仅可见自己的私藏书，游客不可见私藏书。
  • 只保留 can_view_book() 作为单本可见性判断，不引入同义函数。

方案

1. 按图书 scope 选择资源源地址

由 SimpleBookFormatter 同时接收 CDN 地址与应用 API 地址。图书为 private 时，封面和缩略图使用应用地址；其他图书继续使用 CDN 地址。所有调用方，包括普通列表与管理员图书列表，都走同一选择逻辑。

BookFormatter.get_files() 对格式下载链接应用同样规则，避免详情页封面修好但下载仍因跨域缺少 Cookie 而 404。

2. 非 JSON 阅读入口保留 404

BookRead、EpubReader 与 BookDownload 统一调用 get_book_or_404()。该助手通过中央 get_book() 复用唯一的 can_view_book() 授权，拒绝或不存在时抛出 HTTP 404；JSON 接口仍保留原有的 not_found 语义。

兼容性、安全与回滚

  • 无配置 static_host 时，应用源与 CDN 地址都为空前缀，仍生成相对 URL，行为不变。
  • 私藏资源不再经过独立静态域名，避免 Cookie 缺失；代价是私藏封面不使用 CDN 缓存，这是授权正确性所需。
  • 公开图书 URL 不变，CDN 命中率不受影响。
  • 不修改数据库、配置字段或前端代码。回滚时可单独撤销本次 follow-up 提交，不影响原权限方案。
